DATABASE REQUIREMENTS
================================================================================

Database System:
  • SQL Server 2016 or higher
  • Express Edition is sufficient for development/small deployments
  • Standard or Enterprise for large deployments

Database Configuration:
  Database Name: ITInventoryDB
  Authentication: Windows Authentication (Trusted Connection)
  Compatibility Level: 130 or higher

Required ODBC Driver:
  ODBC Driver 18 for SQL Server
  Download: https://learn.microsoft.com/en-us/sql/connect/odbc/download-odbc-driver-for-sql-server

Connection String Format:
  DRIVER={ODBC Driver 18 for SQL Server};SERVER=localhost;DATABASE=ITInventoryDB;Trusted_Connection=yes;TrustServerCertificate=yes;

Alternative Connection (SQL Authentication):
  DRIVER={ODBC Driver 18 for SQL Server};SERVER=servername;DATABASE=ITInventoryDB;UID=username;PWD=password;

ENVIRONMENT VARIABLES (.env file)
================================================================================

Create a .env file in the backend/ folder with these variables:

Security:
  SECRET_KEY=your-secret-key-here
    - Used for JWT token signing
    - Change in production to a secure random string

  ACCESS_TOKEN_EXPIRE_MINUTES=60
    - JWT token expiration time in minutes

Database:
  DATABASE_URL=DRIVER={ODBC Driver 18 for SQL Server};SERVER=localhost;DATABASE=ITInventoryDB;Trusted_Connection=yes;TrustServerCertificate=yes;
    - Connection string for SQL Server database

Email (Optional, for notifications):
  SMTP_SERVER=smtp.gmail.com
    - SMTP server address

  SMTP_PORT=587
    - SMTP server port

  SMTP_USER=your-email@gmail.com
    - Email account for sending notifications

  SMTP_PASSWORD=your-app-password
    - Email account password or app-specific password

================================================================================
TROUBLESHOOTING REFERENCE
================================================================================

Issue: ModuleNotFoundError
Solution: 
  1. Ensure virtual environment is activated
  2. Run: pip install -r requirements.txt
  3. Verify Python version: python --version

Issue: npm ERR! code EACCES (permission denied)
Solution:
  1. Don't use sudo with npm
  2. Fix npm permissions: npm config set prefix '~/.npm-global'
  3. Add to PATH: export PATH=~/.npm-global/bin:$PATH

Issue: ODBC Driver not found
Solution:
  1. Download ODBC Driver 18 from Microsoft
  2. Run installer as Administrator
  3. Restart application after installation

Issue: SQL Server connection refused
Solution:
  1. Verify SQL Server service is running
  2. Check server name in connection string
  3. Test with sqlcmd: sqlcmd -S localhost -E

Issue: Port already in use
Solution:
  1. Find process using port (Windows): netstat -ano | findstr :8000
  2. Kill process: taskkill /PID <PID> /F
  3. Or use different port (see PORT REQUIREMENTS section)
